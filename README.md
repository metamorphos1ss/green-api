# GREEN-API Test Task

## Код

Чистый HTML/CSS/JS, три файла, без сборки. Задание на статическую страницу — тащить туда реакт незачем.

Весь JS — это один хелпер `request()` плюс четыре обработчика кнопок. Хелпер берёт метод, путь и опциональное тело, делает fetch, кладёт результат в textarea. GET-ы вызывают его без тела, POST-ы — с. Больше абстрагировать нечего.

Переменные: `const` для того что объявил один раз и забыл, `var` внутри функций. `let` не использую — привычка.

Ошибки падают прямо в textarea через `err.message`. Отдельного блока для ошибок нет и не нужно.

CSS написан руками. Двухколоночный грид: левая `380px`, правая растягивается. Textarea справа на всю высоту через `flex: 1`. Цвета с сайта грин-апи, значения подобраны на глаз.

## Деплой

GitHub Pages показался слишком базовым для портфолио, поэтому поднял на своём железе. В Proxmox создал виртуалку (Ubuntu 24.04, 1 ядро, 2GB RAM), положил файлы в `/var/www/green-api`, симлинк из `infra/nginx/green-api.conf` в `sites-enabled` — nginx отдаёт статику.

Сеть устроена так: снаружи VPS с reverse proxy, дальше трафик идёт через Tailscale с ACL до домашнего nginx, который уже раздаёт по нужным виртуалкам. 22 порт наружу не торчит намеренно, и лишних не прокидываю.

Это и сломало изначальную идею с CD: хотел сделать деплой через SSH из GitHub Actions — скрипт `deploy.sh` просто делает `git pull` и `nginx -s reload`. SSH снаружи недоступен, пробрасывать ничего не хотелось, поэтому сделал по-другому.

В `infra/webhook/` лежит маленький HTTP-сервер на питоне — слушает на `127.0.0.1:9000`, проверяет Bearer-токен, запускает `deploy.sh`. Nginx проксирует на него `/webhook`. GitHub Actions при пуше тега делает туда curl с секретом из Secrets. Параллельно второй job деплоит на GitHub Pages.

Итого два места: [green-api.metairony.dev](https://green-api.metairony.dev) и GitHub Pages.
